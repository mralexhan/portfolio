A Journey in Transit

2023

Skills Used: UX Design, UI Design, Collaboration, User Research, Market Research, Wireframing, Prototyping, User Testing, Frontend Web Development, Animation

A Journey in Transit was a UX focused design project that aimed to spread public awareness of Indigenous cultures within Sydney. To achieve this goal, our group designed an interactive bus stop as well as an AR enhanced bus ride that would teach commuters little tidbits of cultural information as they passed through different areas of Sydney.

This design was informed by in-depth primary research of various stakeholders held through questionnaires, pop-ups, and focus groups, as well as market analysis of other solutions tackling the same problem space.

This research was followed up by the creation of mid-fidelity physical prototypes of the bus stop and AR bus windows created through wood and cardboard. We used this prototype to conduct several rounds of user testing, where we gained valuable feedback on bus stop screen UI, spatial considerations for the user, and overall effectiveness of the concept.

The project was capped with the creation of a to scale high-fidelity prototype of the interactive bus stop screen and AR bus window, achieved through projectors, a simple web app, and custom made animations.

Visualising Biodiversity

2023

Skills Used: Design Communication, Data Visualisation, Data Organisation, Wireframing, Mockups, Prototyping, User Testing, Frontend Web Development

Visualising Biodiversity was a design project mixing data visualisation and communication design. In this project, I chose to highlight the effects food agriculture has on the terrestrial biodiversity of Australia to help people understand how foods they eat may affect the wildlife of the country.

For the data visualisation, line graphs were used to highlight the correlation between food agriculture and biodiversity. An interactive choropleth map was also created so users can see areas where farmland of different food products overlap with the habitats of threatened wildlife. The data was found on publicly available sources and collated and organised in Microsoft Excel.

In order to present the data and ideas, a simple website was created through HTML and CSS, with the interactive graphs being programmed through the Plotly.js library. Careful attention was to the usability and readability of the website, with user testing being conducted to make sure the core concept was being properly communicated.

ReMi

2022

Skills Used: UI Design, Collaboration, User Research, Wireframing, Mockups, Prototyping, User Testing

ReMi was a UI focused design project that aimed to help students better organise their study materials, whether they be in-class (ex.textbooks) or out-of-class resources (ex. YouTube videos). The concept consists of an online library where users would be able to easily find and save resources they find to be useful in a extendable sidebar.

The design was first ideated in sketches, then fleshed out in wireframes and mockups. To test the usability of the website, as well as the overall effectiveness of the concept, a click-through prototype was created through Figma, which was then subject to various usability tests.

These usability tests included both expert-based heuristic testing and standard user testing. The results of these tests were collated and synthesised into actionable changes that could be made to the website. Several rounds of usability tests were performed to trim out or fix unnecessary or clunky features.

Studbud

2022

Skills Used: Frontend Web Development, UI Design, User Research, Wireframing, Prototyping, Mockups

StudBud was a Frontend Web Design project focused on helping students manage their study time. The project consisted of designing, then programming a web application from the ground up. First, the functionalities of the web app were decided upon. These were informed by the results of user research into the selected target audience in the form of surveys and online ethnographies conducted. Next, sketches and wireframes were made that fleshed out the details of the layout and defined the flow of the web app. Finally, a mockup was created that finalised the web app's colours and visuals.

Before the programming began, a functional specification sheet was created to detail which functions needed to be on the web app and prioritise their importance. This was crucial in deciding how to split up the work in the limited time that was given. I began with coding the look of the web app through HTML and CSS, keeping responsive web design in mind so the app could work on both desktop and mobile. I then programmed the functionalities of the web app through Javascript, which included a kanban board, a pomodoro timer, a music player, and local storage capabilities.
